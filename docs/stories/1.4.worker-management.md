# Story 1.4: Worker Management

## Status

Draft

## Story

**As a** factory supervisor,
**I want** to manage worker records,
**so that** I can track production and calculate wages.

## Acceptance Criteria

1. Worker schema with name, type (weaver/warper/pasariwala), factoryId, rate, isActive
2. GET `/workers` with type and factory filters
3. POST `/workers` creates worker
4. PUT/DELETE endpoints for workers
5. Rate field used for salary calculation

## Dev Notes

### Worker Schema

```typescript
{
  name: string,
  type: 'weaver' | 'warper' | 'pasariwala' | 'jobworker',
  factoryId: ObjectId,
  rate: number,           // per unit rate
  rateUnit: 'meter' | 'taka' | 'beam',
  phone: string,
  isActive: boolean
}
```
